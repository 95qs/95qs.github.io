题目：

139. 单词拆分 - 力扣（LeetCode）

自己的解法：

代码1(求出所有可能)：

这种求出所有可能的代码，一定是超时的，其实我们只需要知道能不能有满足的可能就行。

我实现的时候是使用的空间优化版（空间优化版的第二个维度存放的是所有可能），分析是按照i是物品j是空间的方式分析的。

dp数组含义：dp[i][j]的含义是0-i的物品任选放到j大小空间的包中，所有的取法。

递推公式：对于当前物品i分为，取和不取两种可能，然后取两种可能的并集；

取（前提是当前字符串能对应上s对应的部分）：那就是dp[i][j] == dp[i-1][j - 字符串i的长度]的所有取法 + 后面加上当前字符串

不取：dp[i][j] == dp[i-1][j]

初始化：需要初始化第一列为空字符串“”。

这里的遍历顺序一定是先空间后物品（也就是组合），因为单词（物品）出现的先后出现的顺序有要求。

复杂度分析：

时间复杂度：O(n * m * (2^(n) + 2^(n-1) …… + 2^(1)))，其中n是s的长度，m是wordDict的规模。考虑s全由字母a组成，单词表为{a, aa, ……, a * n}，那么第三个循环其实是对于s前i个长度的子字符串有多少种分割方式（已知每个位置都有两种情况分割或者不分割，所以对于当前长度i，分割情况有2^(i)种）。

空间复杂度：O(n * (2^(n) + 2^(n-1) …… + 2^(1)))。

代码2（只判断是否可以）：

dp数组含义：dp[i][j]的含义是0-i的物品（单词）任选放到j大小空间（s的子字符串0-j）的包中，能否构成s的子字符串0-j。

递推公式：对于当前物品i分为，取和不取两种可能，然后取两种可能“或”；

取（前提是当前字符串能对应上s对应的部分）：那就是dp[i-1][j - 字符串i的长度] == true，则dp[i][j] == true

不取：dp[i][j] == dp[i-1][j]

初始化：需要初始化第一列为true，因为空字符串“”一定能构成（直接全部单词都不取）。

这里的遍历顺序一定是先空间后物品（也就是组合），因为单词（物品）出现的先后出现的顺序有要求。

复杂度分析：

时间复杂度：O(n * n * n)，其中n是s的长度，m是wordDict的规模。之所以是n * n是因为需要截取子字符串以及字符串比对，考虑字符串全是a的情况，单词表是{a, aa, ……, a*n}， 这样里面的for的复杂度是一个累加和，复杂度为O(n * n)，总体的复杂度就是O(n^3)。

空间复杂度：O(n)。

代码随想录提供了两种解法：一种是回溯算法，把每种分割情况都给枚举出来（同时为了避免重复计算，额外的空间缓存信息）；第二种是动态规划，这种解法与我们的解法2一样。这里就只粘贴了回溯算法的代码，供学习一下。
